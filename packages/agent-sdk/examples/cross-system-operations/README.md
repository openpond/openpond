# Cross-System Operations

This first-party Agent SDK project is the normal Make Agent/import target for the Cross-System Operations specialist. Import this directory as a local project; it is detected through its `openpond-agent-sdk` dependency and appears in the ordinary desktop project selector.

The project owns the frontier model instructions and the four production-shaped read-only tool contracts. It intentionally owns no customer records and stores no credentials. A generated Taskset environment supplies the versioned synthetic CRM, billing, support, Python state, attempt budgets, and frozen verifier. Both sides must report the same `toolContractHash` before a run starts.

The default chat action uses the normal OpenPond-managed frontier binding and exposes only:

- `search_crm`
- `query_billing`
- `search_support`
- `run_python`

Direct tool execution without a bound synthetic Taskset returns a typed configuration error; it never falls through to a production integration.
